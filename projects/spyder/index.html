Infrastructure / Spyder

Trusted nodesPacket meshBuilder cockpitNo distributed execution claim

The shared substrate under future Manna systems.

Spyder is a local-first trusted-node project for file-native packets, reviewed work handoff, local builder cockpit runs, and safe coordination between machines.

It has proven two-node packet replication and manual work-result return, but it should still be described honestly: coordination and reviewed handoff, not autonomous distributed compute.

Current proof

Two-node packet replication, dashboard mesh status, local builder result exports, and manual reviewed work handoff.

Boundary

Private overlays, secrets, and personal Manna memory do not become shared substrate data.
