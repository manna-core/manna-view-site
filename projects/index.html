Truth Rule

Public profile is not the same as public release

This site is being updated to show the real project network more honestly. Some entries are public pages, some are internal software systems, and some are detailed public-facing briefs for projects that are still in active development.

Network Rules

What the public site should communicate clearly

Status Matters

Each project should read truthfully as live, internal, prototype-stage, or planning-stage instead of pretending everything is launch-ready.

One Network

The site should make the ecosystem feel connected even when the projects have very different surfaces, tones, and audiences.

Public With Intent

Only the right surface should be public: storefronts where helpful, project profiles where useful, and private tools where they still belong.

The project network is meant to keep growing from here: more public project profiles, stronger subpage structure, and eventually dedicated app or subdomain routes where they actually make sense.
