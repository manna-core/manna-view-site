# Website Decision Memory

- `manna-core.dev` is the umbrella home for the broader manna ecosystem, not only a `Manna-View`
  landing page anymore.
- public profile is not the same as public release; status language should stay explicit.
- `Manna-Landscape` should keep reading like a customer-facing business surface, not like a sibling
  app-marketing page.
- until there is a stronger real photo library, `Manna-Landscape` should avoid thin gallery
  sections and lean on structure, trust cues, and a strong quote flow instead.
- do not invent fake social proof, inflated claims, or generic contractor hype just to make the
  page feel more successful.
- the shared site style should feel aligned with the `manna` design DNA: polished, tactical,
  slightly futuristic, and not generic SaaS.
- detailed project pages should be added selectively where they clarify the network, not because
  every internal build needs its own hype page.
