# Website Current State

## Summary

`D:\Manna-core\projects\Website` is the canonical static source for `manna-core.dev`.

It currently powers:

- the root homepage
- the shared `Projects` hub
- the `Deep Delvers` profile
- the `Manna-View` page
- the separate `Manna-Landscape` route

## Latest Batch Outcome

- the shared public-site UI has been refreshed with a cleaner visual system, stronger hierarchy,
  and more professional presentation
- the root homepage now reads like a true public front door for the manna ecosystem instead of a
  lightly edited internal summary
- the `Projects` hub now presents public routes, internal build lanes, and status rules more
  clearly
- the `Manna-View` page copy is now tighter and more product-shaped while keeping honest
  boundaries
- the shared page shell now feels more consistent across root, project hub, and detailed project
  routes
- the `Manna-Landscape` page has now been rebuilt into a stronger local-service business surface:
  cleaner hero, clearer business copy, no photo carousel for now, stronger quote-first structure,
  and a more professional service-company feel overall

## Current Strengths

- static hosting remains simple through GitHub Pages
- the site now has clearer public language and better visual cohesion
- route roles are easier to understand at a glance
- the shared style feels more aligned with the `manna` design DNA

## Current Gaps

- the refreshed files still need to be published to the live Pages repo to change the public site
- social preview metadata and favicon handling are still lightweight
- the public site still has no stronger deployment checklist or publish script in this project
- `Manna-Landscape` still has limited proof assets, so the site should keep leaning on clear
  structure and honest service language instead of pretending there is a deep media library or
  mature review system already in place

## Active Direction

Keep `manna-core.dev` feeling like one intentional network:

- root page as umbrella identity
- project hub as truthful map
- detailed profiles only where helpful
- customer-facing routes with their own voice when needed
- business routes like `Manna-Landscape` should feel like real service-company sites, not internal
  project pages with a contact form attached
